## 1. 项目概述

### 1.1 项目背景

大富翁（Monopoly）是一款经典的多人策略桌游。本项目旨在将其移植为纯本地运行的网页版本，玩家可以在同一台设备上通过浏览器进行 2-4 人的本地对战。

### 1.2 项目目标

- **完整还原**经典大富翁游戏体验，包括地产交易、房屋/酒店建造、机会/命运卡、监狱、税收等核心机制
- **纯前端实现**，零后端依赖，打开浏览器即可游玩
- **响应式设计**，适配桌面端和平板端
- **良好的代码架构**，模块化、可扩展、易维护

### 1.3 技术选型

| 技术 | 选型 | 说明 |
|------|------|------|
| 框架 | React 18 | 组件化开发，生态成熟 |
| 语言 | TypeScript | 类型安全，提升代码质量 |
| 构建工具 | Vite | 快速的开发体验和构建速度 |
| 状态管理 | Zustand | 轻量级，API 简洁，适合游戏状态 |
| 样式方案 | CSS Modules + Tailwind CSS | 模块化样式 + 实用工具类 |
| 动画 | Framer Motion | 流畅的游戏动画效果 |
| 本地存储 | localStorage | 游戏存档与设置持久化 |
| 测试 | Vitest + React Testing Library | 单元测试与组件测试 |

### 1.4 约束条件

- 纯本地运行，不依赖任何后端 API 或云服务
- 单 HTML 页面应用（SPA）
- 支持 Chrome、Firefox、Safari、Edge 等现代浏览器
- 不支持移动端竖屏操作（推荐横屏或平板）

## 2. 游戏规则与需求分析

### 2.1 游戏基本信息

| 属性 | 值 |
|------|-----|
| 玩家人数 | 2-4 人（本地热座） |
| 棋盘格数 | 40 格 |
| 初始资金 | ¥1500 |
| 起始位置 | GO（第 0 格） |
| 骰子 | 2 颗六面骰 |

### 2.2 棋盘布局

棋盘为经典正方形布局，每边 11 格（含角落），共 40 格。四个角落分别为：

| 位置 | 格子名称 | 功能 |
|------|----------|------|
| 0 | GO | 经过或停留获得 ¥200 |
| 10 | 监狱 / 探视 | 被关押或探视 |
| 20 | 免费停车 | 无特殊效果（可选：收取罚款池） |
| 30 | 前往监狱 | 被送入监狱 |

| 橙色 | 3 | ¥180-¥220 | ¥14-¥180 | ¥100 |
| 红色 | 3 | ¥220-¥260 | ¥18-¥220 | ¥150 |

### 2.4 核心游戏流程

```
游戏开始
  ↓
[设置界面] → 选择玩家人数、玩家名称、头像颜色
  ↓
[游戏主界面]
  ↓
轮流回合制：
  ┌─────────────────────────────────┐
  │ 1. 掷骰子（自动或点击）          │
  │ 2. 移动棋子到目标格              │
  │ 3. 触发格子事件：                │
  │    ├─ 空地/他人地产 → 可购买/付租 │
  │    ├─ 自己地产 → 无事发生        │
  │    ├─ 机会/命运 → 抽卡执行       │
  │    ├─ 税收 → 扣钱               │
  │    ├─ 前往监狱 → 入狱           │
  │    └─ GO → 收钱                 │
  │ 4. 可选操作：                    │
  │    ├─ 建造房屋/酒店              │
  │    ├─ 与其他玩家交易             │
  │    └─ 抵押/赎回地产              │
  │ 5. 回合结束，轮到下一位玩家       │
  └─────────────────────────────────┘
  ↓
[游戏结束] → 最后一位未破产的玩家获胜
```

### 2.5 关键规则详解

#### 2.5.1 掷骰子

- 每回合掷 2 颗六面骰，点数之和决定移动步数
- **掷出双数**：
  - 再掷一次（连续行动）
  - 连续掷出 3 次双数 → 直接送入监狱
- **在监狱中**：
  - 掷出双数 → 立即出狱，按骰子点数移动
  - 支付 ¥50 → 出狱，正常掷骰
  - 使用"出狱卡" → 出狱，正常掷骰
  - 最多关押 3 个回合，第 3 回合强制支付 ¥50 出狱

#### 2.5.2 购买地产

- 玩家到达无主地产时，可选择购买或拍卖
- 购买价格即为地产标价
- 资金不足时无法购买，地产进入拍卖流程
- **拍卖规则**：所有玩家参与竞价，起拍价为地产价格的 10%，每次加价至少 ¥10，最高出价者获得

#### 2.5.3 支付租金

- 玩家到达他人拥有的地产时，需支付租金
- **基础租金**：根据地产卡片上的租金表
- **拥有完整颜色组**：租金翻倍
- **建造房屋/酒店**：租金大幅提升（按卡片租金阶梯表）
- 资金不足时，需选择：出售建筑、抵押地产、破产

#### 2.5.4 建造房屋与酒店

- 前提条件：
  - 拥有该颜色组的**全部**地产
  - 当前不在监狱中
  - 建造资金充足
- 规则：
  - 每次只能在一个颜色组上建造
  - 必须**均匀建造**：先在每块地上各建 1 栋，才能建第 2 栋
  - 每块地产最多建造 **4 栋房屋 + 1 座酒店**（酒店替代 4 栋房屋）
  - 建造顺序：可在任意回合内建造，不限次数（资金允许）

#### 2.5.5 抵押与赎回

- **抵押**：将地产抵押获得现金（地产价格的 50%）
- 抵押期间：
  - 不收租金
  - 不能建造房屋
  - 棋子仍正常经过
- **赎回**：支付抵押金额 + 10% 利息赎回

#### 2.5.6 破产

- 玩家无法支付所需费用时宣告破产
- 破产玩家的所有资产转移给债权人：
  - 地产（含建筑）
  - 抵押中的地产（债权人支付赎回金）
  - 机会/命运卡
- 如果欠银行，所有资产归还银行（从游戏中移除）
- 最后一位未破产的玩家获胜

#### 2.5.7 玩家间交易

- 在任意玩家回合内，当前玩家可与其他玩家发起交易
- 交易内容可包括：
  - 地产（含/不含建筑）
  - 现金
  - 机会/命运卡
  - "出狱免费卡"
- 双方同意后交易生效

### 3.3 模块职责

| 模块 | 职责 | 关键接口 |
|------|------|----------|
| `components/` | UI 渲染与用户交互 | React 组件 Props |
| `stores/` | 全局状态管理与状态变更 | Zustand Store Actions |
| `logic/` | 纯函数业务逻辑，无副作用 | 输入游戏状态 → 输出结果 |
| `data/` | 静态配置数据 | 导出常量数组 |
| `types/` | TypeScript 类型定义 | Interface / Type |
| `hooks/` | 封装可复用的状态逻辑 | Custom Hook |

### 3.4 设计原则

1. **逻辑与视图分离**：`logic/` 层为纯函数，不依赖 React，便于测试
2. **单向数据流**：UI → Store Action → State 更新 → UI 重渲染
3. **不可变数据**：所有状态更新返回新对象，不修改原状态
4. **事件驱动**：游戏事件（掷骰、购买、建造等）通过 Store Action 触发

 * 计算租金
 * @param propertyId 地产 ID

  diceValues: [number, number]

 * 判断玩家是否拥有完整颜色组
 */

  playerId: PlayerId,
  colorGroup: ColorGroup,
  gameState: GameState
): boolean;

**租金计算规则**：

```
if (地产已抵押) → 租金 = 0
else if (有酒店) → 租金 = rentByHouse[5]
else if (有 N 栋房屋) → 租金 = rentByHouse[N]
else if (拥有完整颜色组) → 租金 = baseRent × 2
else → 租金 = baseRent

if (铁路):
  拥有 N 条铁路 → 租金 = 25 × 2^(N-1)

if (公用事业):
  拥有 1 家 → 租金 = 骰子总和 × 4
  拥有 2 家 → 租金 = 骰子总和 × 10
```

### 8.2 状态更新流程示例

**掷骰子 → 移动 → 付租 完整流程**：

```
1. 用户点击"掷骰子"
   → UIStore.setDiceRolling(true)
   → 播放骰子动画

2. 动画完成
   → GameStore.rollDice()
     ├─ 生成随机骰子值
     ├─ 判断双数
     ├─ 更新 doublesCount
     └─ 若连续3次双数 → sendToJail()

3. 骰子结果显示
   → UIStore.setDiceRolling(false)
   → 开始移动动画

4. 移动动画完成
   → GameStore.movePlayer()
     ├─ 计算新位置
     ├─ 判断是否经过 GO → 收钱
     └─ 更新玩家位置

5. 触发格子事件
   → 根据格子类型执行不同逻辑
   → 例如：到达他人地产
     ├─ calculateRent()
     ├─ 扣除付款方金额
     ├─ 增加收款方金额
     ├─ 添加游戏日志
     └─ UIStore.openModal('card_effect', { ... })

6. 玩家选择后续操作
   → 建造 / 交易 / 结束回合

7. 结束回合
   → GameStore.endTurn()
     ├─ 重置 doublesCount
     ├─ 切换到下一位活跃玩家
     └─ 更新回合数
```

### 12.3 术语表

| 术语 | 英文 | 说明 |
|------|------|------|
| 颜色组 | Color Group | 同一颜色的 2-3 块地产构成一组 |
| 抵押 | Mortgage | 将地产抵押给银行获取现金 |
| 赎回 | Unmortgage | 支付本金+利息赎回抵押地产 |
| 拍卖 | Auction | 玩家竞价购买无人购买的地产 |
| 双数 | Doubles | 两颗骰子点数相同 |
| 热座 | Hot Seat | 多人共用同一设备的游戏模式 |
| 完整组 | Full Set | 拥有某颜色组的全部地产 |
